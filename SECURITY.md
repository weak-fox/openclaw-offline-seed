# Security Policy

## Supported Versions

Security fixes are applied on a best-effort basis to actively maintained
branches.

| Version | Supported |
| --- | --- |
| `main` | :white_check_mark: |
| Other branches/tags | :x: |

## Reporting a Vulnerability

Please do not open a public GitHub issue for security reports.

Report vulnerabilities privately to:
`827367480@qq.com`

Please include:

- Affected version/commit
- Impact assessment
- Reproduction steps or proof of concept
- Any proposed mitigation (optional)

## Response Expectations

- Initial acknowledgment: within 3 business days
- Triage update: within 7 business days
- Fix timeline: depends on severity and reproducibility

If the report is accepted, we will coordinate disclosure timing and release
notes with you.
